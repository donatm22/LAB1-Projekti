USER ROUTES REQUEST BLUEPRINTS

Base path: /users

1) Get all users
Method: GET
Endpoint: /users
Body: none

Example request:
GET /users

Example response:
[
  {
    "id": 1,
    "emri": "John Doe",
    "email": "john@example.com",
    "roli": "user"
  }
]

2) Get user by id
Method: GET
Endpoint: /users/:id
Body: none

Example request:
GET /users/1

Example response:
{
  "id": 1,
  "emri": "John Doe",
  "email": "john@example.com",
  "roli": "user"
}

3) Register user
Method: POST
Endpoint: /users/create
Body:
{
  "emri": "John Doe",
  "email": "john@example.com",
  "password": "secret123",
  "roli": "user"
}

Example request:
POST /users/create

Example response:
{
  "message": "User u shtua me sukses",
  "user": {
    "id": 1,
    "emri": "John Doe",
    "email": "john@example.com",
    "roli": "user"
  }
}

4) Update user
Method: PUT
Endpoint: /users/update/:id
Body:
{
  "emri": "John Doe",
  "email": "john@example.com",
  "password": "secret123",
  "roli": "admin"
}

Example request:
PUT /users/update/1

Example response:
{
  "message": "User u perditesua me sukses"
}

5) Delete user
Method: DELETE
Endpoint: /users/deleteUser/:id
Body: none

Example request:
DELETE /users/deleteUser/1

Example response:
{
  "message": "User u fshi me sukses"
}

AUTH REQUEST BLUEPRINTS

Base path: /auth

6) Login
Method: POST
Endpoint: /auth/login
Body:
{
  "email": "john@example.com",
  "password": "secret123"
}

Example request:
POST /auth/login

Example response:
{
  "message": "Login successful",
  "token": "jwt-token-here",
  "user": {
    "id": 1,
    "emri": "John Doe",
    "email": "john@example.com",
    "roli": "user"
  }
}

Notes:
- The route file is mounted under `/users` in `server.js`.
- Passwords are stored with bcrypt hashing on register.
- Login uses Passport local strategy and returns a JWT token.
